# Movement Test PRD (최소)

## 목적
사용자가 /movement-test 진입 후 설문을 끝까지 완료하고, /movement-test/result에서 결과를 즉시 확인하도록 만든다.
설문 이탈률을 줄이기 위해 "진행 저장/복구"와 "명확한 이전/다음 UX"를 보장한다.

## 유저 플로우
1) /movement-test (소개) → "테스트 시작" 클릭
2) /movement-test/survey (설문 40문항, 페이지네이션)
3) 완료 시 /movement-test/result 이동 및 결과 표시

## 성공 조건 (KPI가 아니라 기능 합격 기준)
- 새로고침/앱 종료 후 재진입해도 설문 진행상태가 복구된다.
- 이전/다음이 항상 정상 동작하고, 답변이 사라지지 않는다.
- 마지막 제출 시 결과 타입이 항상 1개로 확정된다(동점 처리 규칙 적용).
- 결과 페이지는 설문 데이터가 없으면 안내 후 /movement-test로 유도한다.
